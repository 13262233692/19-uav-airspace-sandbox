## 核心技术特性

### 后端 (Node.js)
- **高性能模拟引擎**: 60Hz 频率模拟 5000 架无人机的完整飞行物理
- **6 种任务类型**: 物流配送、巡逻监控、设施巡检、应急响应、测绘建模、载人飞行
- **航点导航系统**: 每架无人机拥有独立的多航点飞行路径
- **二进制数据协议**: 高效的二进制编码，单帧仅约 160KB (5000 架 × 32 字节)
- **30Hz 实时广播**: 低延迟 WebSocket 数据流分发

### 前端 (Three.js + WebGL)
- **InstancedMesh 渲染**: 5000 架无人机仅需 **1 次绘制调用** (Draw Call)
- **GPU 着色器动画**: 旋翼旋转、姿态解算、飞行抖动完全在 GPU 层级执行
- **自定义实例属性**: 通过 `InstancedBufferAttribute` 传递每架无人机的任务类型、速度、ID
- **程序化城市生成**: 5 个功能分区，数百栋建筑，合并几何优化性能
- **空域可视化**: 5 层高度管制空域可视化

## 性能指标

| 指标 | 目标值 | 实现方式 |
|------|--------|----------|
| 无人机数量 | 5000 | InstancedMesh 批量渲染 |
| 绘制调用 | 1 | 单 InstancedMesh 实例 |
| 模拟频率 | 60 Hz | Node.js setInterval |
| 数据更新 | 30 Hz | WebSocket 二进制广播 |
| 渲染帧率 | ≥ 60 FPS | GPU 动画解耦 |
| 数据带宽 | ~38 Mbps | 二进制协议 + 批量传输 |

## 数据协议

### 二进制帧格式 (Little Endian)

| 偏移 | 类型 | 描述 |
|------|------|------|
| 0 | uint32 | 无人机数量 (UAV Count) |
| 4 | uint32 | 时间戳 (Timestamp, ms) |
| 8 | float32[] | 无人机状态数组 |

### 单架无人机状态 (32 bytes, 8 × float32)

| 索引 | 字段 | 描述 |
|------|------|------|
| 0 | x | X 坐标 (东向) |
| 1 | y | Y 坐标 (高度) |
| 2 | z | Z 坐标 (北向) |
| 3 | heading | 航向角 (弧度) |
| 4 | speed | 当前速度 (m/s) |
| 5 | missionType | 任务类型 ID (0-5) |
| 6 | vx | X 方向速度分量 |
| 7 | vz | Z 方向速度分量 |

## GPU 着色器特性

### 顶点着色器 (Vertex Shader)
- **旋翼旋转动画**: 基于 `uTime` 和速度因子在 GPU 上实时计算
- **交错旋转方向**: 相邻旋翼反向旋转，模拟真实四旋翼气动特性
- **飞行抖动**: 基于 `sin(uTime + aUAVId)` 的细微高度变化
- **实例矩阵变换**: `instanceMatrix` 应用位置、朝向、倾斜

### 片段着色器 (Fragment Shader)
- **任务类型着色**: 根据 `aMissionType` 索引颜色数组
- **高度辉光**: 飞行高度越高，自发光越强
- **速度辉光**: 速度越快，颜色越亮
- **菲涅尔边缘光**: 基于视线角度的 Rim Lighting
- **大气雾效**: 指数雾模拟距离衰减
- **脉冲呼吸灯**: 每架无人机独立的 ID 偏移脉冲

## 性能优化要点

1. **几何合并**: 城市建筑合并为单个 BufferGeometry
2. **实例化渲染**: 5000 架无人机使用 InstancedMesh，单次绘制
3. **GPU 动画**: 所有动画在 Shader 中计算，无 CPU 开销
4. **动态顶点缓冲区**: `DynamicDrawUsage` 优化频繁更新
5. **视锥体剔除关闭**: `frustumCulled = false` 避免实例化对象错误剔除
6. **二进制传输**: WebSocket 使用 ArrayBuffer，减少解析开销
7. **共享内存**: 后端使用 `SharedArrayBuffer` 优化数据访问
